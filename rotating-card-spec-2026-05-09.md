## 1. 6 Source 정의

### 1-1. 🌟 진주

- 컨텐츠: 사용자 진주 1장 (현 `_pickHeroPearl` + `_heroCardHtml`)
- 회전 빈도: **4시간 같은 진주 고정**. 4시간 후 다른 진주로
- 가용 조건: dna 제외 진주 ≥1
- 빈 상태: 빈 진주 CTA (`_heroEmptyHtml`)
- 컨펌 정의: 사용자가 카드 1번 본 시점 = 컨펌

### 1-2. ✨ 새로 본 너 (case formulation 변경)

- 컨텐츠: 새벽 4시 분석에서 traits / values / patterns / case formulation 에 *새로 추가된 항목 1개*
- 회전 빈도: **새 인사이트 생길 때 바뀜**. 새 항목 X 면 같은 컨텐츠 stay
- 가용 조건: 미컨펌 새 항목 ≥1
- 컨펌 정의: 사용자가 [맞아 ✓] / [아닌데 ✕] 둘 중 하나 누른 시점 = 컨펌. 그 항목 user_verified=true (or down-weight)
- 톤: 친구 화법 ("있잖아, 너 ___ 더라")

### 1-3. 📔 미니 리뷰 (3일 / 주말)

- 컨텐츠: Haiku 1턴 호출로 생성된 *지난 3일 한 단락*
- 회전 빈도: **3일 동안 같은 리뷰 고정**. 다음 trigger 도래 시 새 리뷰 생성
- trigger 조건:
  - (마지막 미니 리뷰로부터 ≥3일) AND (체크인 ≥2 OR chat ≥8 turn OR 미션 ≥1 since)
  - OR 토요일 저녁 18시 이후 + 일요일 정식 주간 리뷰 trigger 전
- 컨펌 정의: 사용자가 카드 본문 탭해서 모달 열어본 시점 = 컨펌. 닫고 다시 와도 같은 리뷰 = 컨펌 유지
- 톤: 친구 단락 ("이 3일 너 — ___")

### 1-4. 💡 통찰 (worth-mentioning 변화)

- 컨텐츠: 새벽 분석에서 detect 된 *어휘 빈도 / 평균 변화 / 모드 카운트* 등 worth-mentioning 변화 1개
- 회전 빈도: **새 통찰 생길 때 바뀜**. detect 안 되면 같은 컨텐츠 stay
- 가용 조건: 미컨펌 새 통찰 ≥1
- 컨펌 정의: 사용자가 [맞아 ✓] / [아닌데 ✕] 누른 시점 = 컨펌
- 톤: 친구 화법 ("어 너 이번 주 ___ 자주 쓰더라")

### 1-5. 🎯 Quiz (고동이 맞히기)

- 컨텐츠: case formulation 의 *user_verified=false* traits / values / patterns 에서 5개 random pick → 한 카드 안 N 질문 묶음
- 회전 빈도: **매일 바뀜** (매일 새벽 4시 cutoff 후 새 5개 random pick) — 단 사용자가 다 답 안 했으면 진행 상태 stash, 이튿날 *남은 질문 + 새로 추가된 항목* 으로 재구성
- 가용 조건: user_verified=false 항목 ≥1 (5개 미만이면 있는 만큼)
- 컨펌 정의: 카드 안 *모든 질문 답한 시점* = 컨펌. 도중에 다른 source 로 넘어가면 진행 상태 stash, 다시 돌아와서 이어서.
- 카드 내부 동작: §5 참조

### 1-6. 🌗 오하아사 (별자리 운세)

- 컨텐츠: 사용자 별자리에 해당하는 *오하아사 일일 운세* 외부 데이터
- 회전 빈도: **매일 바뀜** (매일 새벽 4시 cutoff 후 새 운세 fetch)
- 가용 조건: 사용자 별자리 설정됨 + 오늘 운세 fetch 성공
- 컨펌 정의: 사용자가 카드 1번 본 시점 = 컨펌
- 첫 진입 시: 별자리 선택 onboarding (§7 참조)
- 데이터 source: §7 참조 (결정 필요)

## 2. 우선순위 — 미컨펌 우선

### 2-1. source 분류 (홈 진입 시점)

각 source 를 *미컨펌* 또는 *컨펌* 둘 중 하나로 분류:

```
미컨펌:
  - 그 source 안에 사용자가 안 본 새 컨텐츠가 있음
  - 또는 새 컨텐츠가 있는데 cooldown(4시간 / 3일) 안 임에도 사용자 컨펌 안 함

컨펌:
  - 사용자가 이미 본 컨텐츠가 cooldown 동안 stay 중
```

### 2-2. 정렬 알고리즘

```
1. 미컨펌 source 들을 위로 (1순위)
2. 컨펌 source 들 아래 (2순위)
3. 같은 그룹 안에서는 score 순
   - score = baseWeight (이전 spec 의 7-source weights 재사용)
   - tie-breaker = source id 안정 순서
```

baseWeight (이전 spec 4-3 그대로):

| Source | baseWeight |
|---|---|
| 미니 리뷰 | 100 |
| 새로 본 너 | 80 |
| 통찰 | 60 |
| 진주 (default) | 20 |
| Quiz | 70 (신규) |
| 오하아사 | 50 (신규) |

특수: 일요일 + 정식 주간 리뷰 도착 → 미니 리뷰 baseWeight 200 격상.

### 2-3. 가용 source 0 인 경우

빈 진주 CTA (source 1 의 빈 상태) 단독 표시. 인디케이터 1개.

## 3. 순서 고정 — 세션 룰

### 3-1. "세션" 정의

**세션 시작** = 사용자가 홈 화면에 진입한 시점
**세션 종료** = 사용자가 홈 화면 떠난 시점 (다른 탭 진입 / 앱 닫기 / 화면 전환)

### 3-2. 세션 내 동작

```
홈 진입
   ↓
[정렬 알고리즘 실행 — 미컨펌 우선 + score]
   ↓
정렬된 순서 = state.rotatingCardState.sessionOrder 에 stash
   ↓
사용자가 좌우 화살로 카드 넘김
   ↓
[순서 변경 X — sessionOrder 그대로 유지]
   ↓
사용자가 컨펌 액션 (예: [맞아] 누름) → 그 source 즉시 컨펌으로 표시
   ↓
[순서 X 그대로]  ← 핵심 룰
```

세션 동안 사용자가 어떤 컨펌 액션을 해도, 카드 순서는 *진입 시 정해진 그 순서* 그대로 유지. 사용자 입장에서 *다음으로 넘기는 동안 카드가 사라지거나 순서 바뀌는 일 없음*.

### 3-3. 세션 종료 → 재진입

```
사용자 홈 떠남 (다른 탭 / 앱 닫음)
   ↓
사용자 다시 홈 진입
   ↓
[정렬 알고리즘 재실행]
   ↓
미컨펌 source 들이 위로 (이전 세션에서 컨펌한 것들 자동 아래)
   ↓
새 sessionOrder 로 stash
```

즉 사용자 입장에서:
- 같은 세션 = 카드 순서 보존, 안정감
- 새 세션 = *안 본 거 우선* 으로 재정렬, 신선함

### 3-4. 예시 시나리오

```
[진입 1차]
정렬 결과 = [퀴즈, 새로 본 너, 진주, 오하아사, 통찰, 미니 리뷰]
사용자: 퀴즈 답함 (컨펌) → 화살 넘김 → 새로 본 너 [맞아] 누름 (컨펌) → 화살 넘김 → 진주 봄
이 동안 카드 순서 그대로 유지.

[다른 탭 갔다가 다시 진입 2차]
컨펌됨: 퀴즈, 새로 본 너, 진주
미컨펌: 오하아사, 통찰, 미니 리뷰
정렬 결과 = [미니 리뷰, 통찰, 오하아사, 진주, 새로 본 너, 퀴즈]  ← 미컨펌 위로
새 세션 시작.
```

## 4. 회전 빈도 정리

각 source 의 *컨텐츠 변경 주기*:

| Source | 컨텐츠 변경 주기 |
|---|---|
| 진주 | 4시간 |
| 새로 본 너 | 새 인사이트 생길 때 |
| 미니 리뷰 | 3일 (다음 trigger 도래 시) |
| 통찰 | 새 통찰 생길 때 |
| Quiz | 매일 (4AM cutoff) |
| 오하아사 | 매일 (4AM cutoff) |

이 주기는 *컨텐츠 자체가 바뀌는 주기*. 사용자가 cooldown 안 다시 와도 같은 컨텐츠 노출 (컨펌 후엔 컨펌으로 표시되고 다른 source 미컨펌 우선).

## 5. Quiz 섹션 — 카드 내부 미니 회전

### 5-1. 구조

quiz 한 source = *N 질문 묶음* (보통 5개). 카드 안에 한 질문씩 표시 + 카드 내부 좌우 화살 (또는 답하면 자동 advance).

### 5-2. 카드 layout

```
┌──────────────────────────────────────┐
│ 🎯  고동이가 너 맞히고 있을까?         │
│                                      │
│  [1/5]                               │
│                                      │
│  너 야행성이지?                       │
│                                      │
│  [맞아 ✓]  [아닌데 ✕]  [넘기기 →]    │
│                                      │
│              ●○○○○                    │
└──────────────────────────────────────┘
```

- 상단: source 라벨
- 진행도: [N/5] 표시
- 본문: 질문 1개
- 액션: 3 chip
- 하단 인디케이터: ●○○○○ (5개 중 현 위치)

### 5-3. 액션 동작

| 액션 | 동작 |
|---|---|
| [맞아 ✓] | 그 항목 user_verified=true → 다음 질문 자동 advance + 짧은 토스트 ("고동이 +1") |
| [아닌데 ✕] | 그 항목 down-weight (confidence 50% 감소 + user_verified=false fix) → 다음 질문 자동 advance + 토스트 ("오케이 다시 볼게") |
| [넘기기 →] | 답 안 함 + 다음 질문 advance (해당 항목 상태 변화 X) |

### 5-4. 카드 내부 좌우 화살

카드 안 *작은* 좌우 화살 (또는 인디케이터 옆) — 답한 질문 다시 보거나 안 본 질문으로 미리 넘김 가능. 기본 동작은 *답하면 자동 advance*.

### 5-5. 컨펌 정의

5개 모두 답한 (또는 [넘기기] 한) 시점 = 컨펌.

### 5-6. 진행 상태 stash

사용자가 quiz 도중에 외부 좌우 화살로 다른 source 로 넘어가면:
- 진행 상태 stash (`state.rotatingCardState.quizProgress = { questionIds, currentIdx, answers }`)
- 다음 진입 시 외부 화살로 quiz 카드 다시 오면 *이어서* (현 위치부터)
- 단 4AM cutoff 넘으면 새 5개 random pick (이전 미답 + 새 항목 합산)

### 5-7. 끝 화면

5개 다 끝나면 카드 내용 변경:

```
┌──────────────────────────────────────┐
│ 🎯  고동이 점수                       │
│                                      │
│       [godong-신남]                   │
│                                      │
│   오늘 4개 맞히고 1개 빗나감.         │
│                                      │
│   고동이가 너 알아간 점수             │
│        67% → 71%                     │
│                                      │
│   [좋아]                             │
└──────────────────────────────────────┘
```

누적 정확도 한 칸 오르는 게 dopamine. 사용자가 [좋아] 누르면 컨펌 + 다음 source 자동 cycle.

## 6. 오하아사 source

### 6-1. 첫 진입 별자리 선택

홈 처음 진입 시 사용자 별자리 미설정 상태면 → 별자리 선택 onboarding 1회:

```
┌──────────────────────────────────────┐
│ 🌗  너의 별자리?                      │
│                                      │
│   ♈ 양자리   ♉ 황소자리              │
│   ♊ 쌍둥이   ♋ 게자리               │
│   ♌ 사자자리 ♍ 처녀자리              │
│   ♎ 천칭자리 ♏ 전갈자리              │
│   ♐ 사수자리 ♑ 염소자리              │
│   ♒ 물병자리 ♓ 물고기자리            │
│                                      │
│   [건너뛰기]                          │
└──────────────────────────────────────┘
```

`state.preferences.userZodiac` 에 stash. 변경은 설정에서.

### 6-2. 매일 운세 fetch

매일 새벽 4시 cutoff 후 사용자 별자리에 맞는 *오하아사 일일 운세* fetch.

데이터 source — **결정 필요** (이 spec 에서 미정):
- 옵션 A: 외부 운세 API / RSS
- 옵션 B: 자체 운세 풀 작성 (카피 cost)
- 옵션 C: 매일 1번 Haiku 호출로 generation (운세 톤)

각 옵션의 trade-off:
- A: 라이선스 / 정책 / 안정성 issue 검토 필요
- B: 한 번 작성 후 평생 사용 (가장 저렴) — 단 *매일 새로움* 약함 (반복)
- C: 매일 새로움 + AI 톤 조정 — 비용 사용자당 월 ~$0.030

추천: **B + C 하이브리드** — 별자리별 베이스 풀 50개 (B) + 매일 한 줄 Haiku 변형 (C)

### 6-3. 카드 layout

```
┌──────────────────────────────────────┐
│ 🌗  오늘의 너 (오하아사)               │
│                                      │
│       ♓ 물고기자리                   │
│                                      │
│  오늘은 새 만남보다 익숙한 곁이        │
│  좋은 결. 오후 3시쯤 작은 신호를       │
│  놓치지 마.                          │
│                                      │
│   행운: 따뜻한 차 한 잔                │
│   조심: 충동 결제                     │
│                              ›       │
└──────────────────────────────────────┘
```

운세 톤 — 친구 톤보다 *살짝 신비롭게* OK. 단 분석 / 진단 단어 X.

### 6-4. 컨펌 정의

사용자가 카드 1번 본 시점 = 컨펌.

### 6-5. 별자리 변경

설정 → "별자리" 항목에서 언제든 변경 가능.

## 7. 컨펌 / dismiss 룰 정리

| Source | 컨펌 액션 |
|---|---|
| 진주 | 카드 1번 봄 |
| 새로 본 너 | [맞아] 또는 [아닌데] 누름 |
| 미니 리뷰 | 카드 본문 탭 → 모달 열어봄 |
| 통찰 | [맞아] 또는 [아닌데] 누름 |
| Quiz | 5개 모두 답함 (또는 [넘기기] 함) |
| 오하아사 | 카드 1번 봄 |

**"카드 1번 봄"** 정의: 그 카드가 사용자 viewport 에 표시된 시점 (좌우 화살로 navigate 해서 도달 — 다른 source 카드를 보고 있는 동안은 컨펌 X).

|---|---|
| `sessionOrder` | 현 세션의 source 순서 array. 홈 진입 시 정렬 결과 stash |
| `sessionStartAt` | 세션 시작 timestamp. 홈 떠나면 reset |
| `confirmedSources` | 현 세션 / 누적 컨펌된 source list |
| `quizProgress` | quiz 진행 상태 (`{questionIds, currentIdx, answers}`) |
| `pearlWindowStart` | 진주 4시간 windowing 시작 |
| `lastMiniReviewAt` | 미니 리뷰 마지막 생성 시점 |
| `lastZodiacFetchAt` | 오하아사 마지막 fetch 시점 + 컨텐츠 |

기타:
- `state.preferences.userZodiac` — 별자리 (`'aries'` ... `'pisces'` 또는 `null`)

## 9. Edge cases

| 상황 | 동작 |
|---|---|
| 모든 source 비활성 (신규 user 첫날) | source 1 의 빈 진주 CTA 단독. 인디케이터 1개. swipe 비활성 |
| Quiz 가용 항목 < 5 | 있는 만큼만 (예: 3 항목 → [1/3]). 끝나면 컨펌 |
| Quiz 가용 항목 0 | quiz source 비활성 (case formulation 항목 다 user_verified) |
| 오하아사 별자리 미설정 | 별자리 선택 onboarding 카드 표시 (= source 자체가 onboarding) |
| 오하아사 fetch 실패 | source 비활성. 다음 진입 시 재시도 |
| 미니 리뷰 Haiku 호출 실패 | 카드는 표시 (이전 미니 리뷰 stay), 모달 탭 시 "지금은 못 정리하겠어, 다시" |
| 4AM cutoff 직후 진입 | 새 quiz 5개 / 새 오하아사 fetch / case formulation 분석 결과 반영. 미컨펌 source 들 우선 등장 |
| 사용자가 컨펌 안 하고 닫음 | 그 source 미컨펌 유지, 다음 세션 우선 등장 |
| 같은 세션에서 좌우 화살 여러 번 | sessionOrder 그대로, 위치만 변경 |

## 10. 검증 / 완료 기준

- [ ] 6 source 다 가용 시 진입 시 *미컨펌 source* 가 위 (홈 진입 후 첫 카드)
- [ ] 컨펌 액션 후에도 같은 세션 동안 sessionOrder 변경 X
- [ ] 다른 탭 → 홈 재진입 시 미컨펌 우선으로 재정렬
- [ ] 진주 4시간 같은 진주 stay
- [ ] 미니 리뷰 3일 같은 리뷰 stay
- [ ] Quiz 카드 안에서 5개 미니 회전 (자동 advance + 인디케이터)
- [ ] Quiz 진행 도중 외부 화살로 다른 source → 다시 quiz 오면 이어서
- [ ] 오하아사 첫 진입 별자리 선택 onboarding
- [ ] 4AM cutoff 후 새 quiz / 새 오하아사 fetch
- [ ] 친구 톤 통과 (분석 보고서 톤 X)

## 11. 결정 필요 (이 spec 에서 미정)

1. **오하아사 데이터 source** (옵션 A / B / C — §6-2)
2. **Quiz 회전 빈도** — "매일 바뀜" vs "새 항목 생길 때 바뀜" — 추천: 매일 random pick (fresh 결)
3. **별자리 미설정 사용자** — onboarding 카드를 source 6 자리에 매번 표시 vs 1번 dismiss 가능
4. **quiz 끝 화면** — *고동이 점수 누적 카운터* 표시 위치 (카드 안 vs 나 탭으로 이동)

---

## 12. 안 하는 것 (의도적 제외)

- 어제 비교 source — *회고 결* 페기 (사용자 명시)
- 회상 source (1년 전 등) — *회고 결* 페기
- Surprise / 기념 source — 페기 (이전 spec 11-3 streak 압박 회피와 동일 결)
- 진주 후보 자동 추출 — *AI 자동 추출 정확도 risk* 페기
- 오늘 한 가지 (마이크로 챌린지) — *훈수 톤* 페기
- 심리테스트 (양자택일 → 결과 라벨) — *오하아사로 대체*
- 회전 카드 안 ▾ 더보기 toggle — 페기
- 자동 fade 회전 — 페기 (사용자 화살 만)
