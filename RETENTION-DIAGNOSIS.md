작성: 2026-05-18 · 기반: 코드 베이스(`src/`) 전수 매핑 + soragodong.com 실사용 + CLAUDE.md

## TL;DR — 가장 중요한 3가지

1. **사용자가 지적한 "깨달음 칩이 눈에 안 들어옴"의 진짜 원인은 칩이 아니라 destination이다.** 채팅에서 "✦ 깨달음으로" 저장 → 토스트 "깨달음 홈에 저장됐어 ✦" → 사용자: "깨달음 홈이 어디?" → 그게 사실 홈 탭 위쪽의 **✨ emoji-only 칩**임. 칩이 활성화되기 전엔 라벨조차 없어서 새 사용자에겐 그냥 별표 아이콘. 저장한 11개의 깨달음이 풍부하게 쌓여있지만 사용자가 도달하는 경로가 거의 숨어있음.

2. **나 탭은 갱신 빈도가 주 1~2회인데 시각적으로 "새 게 있다"는 신호가 거의 없다.** `_navBatchUpdated.model` 빨간점 시스템이 있긴 하나(`23-archive/04-category-dot-tracking.js`), 갱신 빈도 자체가 낮아서 사용자가 매일 들어와도 어제와 같은 화면. 카드는 "확신 100%, 근거 4개" 같은 메타 데이터 위주의 정적 표시. "+25, +16" 카운터는 정보를 **숨기고** 있지 **유혹하지** 않음.

3. **다이어리 앱 리텐션의 1번 무기인 streak이 0개 있다.** 코드 전수 검색 결과 `streak`, `연속`, 'streak' 키워드 0건. 대신 "어제 대비 24h 변화" 카드(`10-home/09-yesterday-change-hint.js`)가 있지만 streak처럼 시각적으로 강하지 않음. Snapchat·Duolingo·BeReal·Streaks·Day One·Stoic — 다이어리/저널링 시장에서 streak 없는 앱이 없는 이유가 있음.

## 0. 타겟층에 대한 첫 번째 도전

너가 말한 3개 페르소나:
- A. 다이어리를 쓰는 사람 (능동적 기록자)
- B. 외로운 사람 (AI 대화·위로 갈구)
- C. 일기 쓰기 귀찮은 사람 (수동적/저마찰 선호)

**이 셋은 같은 기능으로 만족시킬 수 없다.** 핵심 동기·진입 마찰·리텐션 트리거가 다 다름.

|     | A. 다이어리   | B. 외로운    | C. 귀찮은   |
|-----|-------------|------------|-----------|
| 핵심 동기 | 기록의 의식, 자기 성찰 | 응답의 즉시성, 정서적 위로 | 마찰 제거, 한 줄로 끝 |
| Day 0 가치 | 캘린더·다시 보기 | 첫 대화의 따뜻함 | "한 줄도 좋아" 인풋 |
| Day 7 리텐션 트리거 | streak·과거 회상 | "소라가 먼저 말을 걸어와" | 푸시·한 줄 응답 |
| Day 30 리텐션 트리거 | 깨달음 누적·통찰 | 관계의 깊이감 | 어느샌가 매일 쓰는 자기 발견 |

지금 앱은 **A에 가장 친절하다(캘린더·아카이브·나 탭 분석). B는 챗 품질 덕분에 만족도는 있을 수 있는데 능동 호출(Hook) 시스템이 18h cooldown으로 잠겨있어 외로움의 즉시성과 안 맞음. C는 거의 케어 안 되어있음** — "한 줄도 좋아" 플레이스홀더는 훌륭하지만 홈 첫 화면에서 한 줄 쓰는 CTA가 없음. 캘린더부터 보임.

**제안: Primary persona를 하나 정하고, 나머지는 합리적 정도로만 지원하는 결정이 필요함.** 지금처럼 셋을 다 잡으려는 디자인은 어느 한 명에게도 1순위 앱이 되기 어려움.

내 추천: **A(다이어리) + C(귀찮은) 하이브리드를 primary로.** 이유:
- B(외로운)는 ChatGPT·캐릭터.ai 등 자본 큰 경쟁자가 압도적
- A+C 조합은 "기록의 의식은 있지만 마찰은 낮은 일기" — 좁고 명확한 포지셔닝
- A는 이미 잘 케어되고 있으니, C 보완에 투자하면 시장 확장
- B는 챗 품질로 자연 흡수 (별도 디자인 필요 X)

## 2. 사용자가 지적한 2가지 문제의 진짜 원인

### 2.1 "깨달음 모음칩이 생각보다 눈에 안 들어온다"

너가 느낀 증상은 맞아. 하지만 **칩 자체의 가시성**만 문제가 아니라, **개념 분산 + Destination 모호성 + Affordance 부재** 3중 문제.

**원인 1: "깨달음"이라는 단어가 3개의 다른 것을 가리킴**
- 채팅 끝 `✦ 깨달음으로` 액션 칩 (`19-chat/02-render-message.js:138-147`) — *행위 트리거*
- 홈의 ✨ emoji 칩 (활성화 후에야 "깨달음" 라벨 등장) — *수집함*
- 아카이브 안의 깨달음 렌즈 (`23-archive/`) — *심화 탐색*

같은 이름인데 다른 위치, 다른 인터랙션. 사용자 멘탈 모델 구축 실패.

**원인 2: 저장 후 destination이 보이지 않음**
실사용 reproduction:
1. 채팅에서 "✦ 깨달음으로" 클릭
2. 토스트: "깨달음 홈에 저장됐어 ✦"
3. **"깨달음 홈"이 어디인가? 사용자가 추론할 단서 0.** 토스트는 2.5초 후 사라짐.
4. 사용자는 저장만 했지 "내 깨달음 모음"을 본 적이 없으니 다시 안 봄.

**원인 3: ✨ 칩의 시각적 weight가 검색바보다 약함**
홈 fold-above 우선순위 (실측 스크린샷):
- 헤더 인사말 + 체크인 (가장 위, 크게)
- "오늘의 너" 카드 (큰 영역)
- 4 emoji 칩 (작은 띠, label 숨김)
- 검색바 + 그리드/리스트 토글 (전체 너비, 활성 색)
- 캘린더 (가장 큰 면적)

칩이 검색바와 캘린더 사이에 끼어 시각적으로 "필터 UI"처럼 보임. **콘텐츠로 들어가는 4개의 main entry**라는 게 안 읽힘.

**원인 4: 깨달음에 대한 "왜 다시 봐야 하나" 동기 부재**
저장된 깨달음 11개에 도달해도, 그 콘텐츠가 **수동적 리스트**로 보임. 토픽 태그(#마법, #대화, #숙고)는 좋은 시작이지만:
- 새 깨달음끼리 연결이 자동 추출되지 않음 (코드상 무관계 카드)
- 시간 흐름·테마 변화 시각화 없음
- "이 깨달음들이 모여 뭐가 됐어"라는 합성된 통찰 없음

**액션:**

| # | 액션 | 임팩트 | 난이도 |
|---|------|-------|--------|
| K1 | 토스트 "깨달음 홈에 저장됐어" 클릭 시 ✨ 칩으로 자동 이동 | 高 | 낮음 (1줄) |
| K2 | 4 칩 라벨 상시 표시 (활성/비활성 무관) — "📔 일기·대화" "🌱 양생방" "✨ 깨달음" "🌀 마법·리뷰" | 高 | 낮음 (CSS만) |
| K3 | 홈 fold-above에 "오늘 새 깨달음 X개" 회전 카드 추가 (Hook 카드 slot 재활용) | 高 | 중간 |
| K4 | 깨달음을 "주제별 컬렉션"으로 자동 묶기 — 같은 토픽 태그 3개+ 모이면 "이번 주 너는 ___에 대해 고민했어" 표지 카드 자동 생성 | 高 | 높음 |
| K5 | "깨달음 → 진주" 또는 "깨달음 → 전략" 발전 경로 명시. 코드상 분리되어 있는 5개 개념(깨달음/전략/진주/통찰/양생)을 사용자 멘탈 모델로 통일 | 中 | 높음 (IA 재설계) |

### 2.2 "나 탭이 재미없음"

증상 정확함. 원인은 두 층위.

**표면 원인: 정적이고 메타데이터 위주**
- 각 카테고리 top 1만 표시, 나머지는 "+25" 카운터로 접힘 → **호기심을 자극하지 않고 노동을 암시**
- "확신 100%, 근거 4개" 라벨이 카드 내부에 노출 → 사용자 입장에서 가치 X (이건 디버그 정보)
- "잘 안 풀릴 때 → 지금은 잘 흘러가는 중" empty state는 정보 가치 0. 위로의 의도지만 페이지 길이만 차지.

**근본 원인: AI 배치 갱신 빈도가 사용자 기대와 맞지 않음**
- 코드: `state.caseFormulation`, `traits/values/patterns` 추출은 주 1~2회 배치
- 사용자 기대: 매일 일기 쓰면 매일 새로운 자기 발견을 보고 싶음
- 격차: 5~6일에 한 번 새 게 도착 → 그동안 들어와도 변화 없음 → "굳이 안 들어감"

**보너스 원인: "+더 깊은 나"가 default-collapsed**
사용자가 가장 흥미로워 할 발달·관계·자기서사 섹션이 접혀있어 첫 인상에서 사라짐.

**액션:**

| # | 액션 | 임팩트 | 난이도 |
|---|------|-------|--------|
| M1 | 나 탭 진입 시 "오늘의 발견" 모듈 추가 — AI가 매일 (배치 외에 매일!) 어제 일기 1개를 골라 traits/values 관점에서 짧은 코멘트 생성 (1줄 비용 vs 1주일 retention) | 매우 高 | 중간 |
| M2 | "확신 100%, 근거 4개" → "근거 4개 보기" 같은 액션 라벨로 변환. 신뢰도는 시각적으로(점 4/4 등) | 中 | 낮음 |
| M3 | "+25" 펼침 카운터를 → "다음에 알아볼까 봐 표시한 25개" 같은 동기 부여 라벨 + 펼치면 새 정보 1개 hilight | 中 | 중간 |
| M4 | "잘 안 풀릴 때" empty state 제거 또는 "지금 안 풀리는 게 있어? 한 줄로 알려줘" 입력 prompt로 전환 (역방향 인입) | 高 | 낮음 |
| M5 | "더 깊은 나"를 default-expanded로 변경 + 빈 필드는 "✎ 한 줄 채우기" 마이크로 task로 변환 (Day 1~30 점진적 자기 입력 → 깊이 누적) | 高 | 낮음 |
| M6 | 나 탭에 "타임라인" 모드 추가 — 각 trait이 언제 처음 발견됐는지, 어떤 사건에서 신뢰도가 올랐는지 시각화. **이게 다이어리 앱만의 무기.** ChatGPT는 못함. | 매우 高 | 높음 |

## 3. 사용자가 지적하지 않은, 더 큰 retention 문제

### 3.1 홈 fold-above에 "오늘 할 일" CTA가 없음

현재 홈은 **회고형**. "오늘의 너"가 어제 데이터를 보여줌. 캘린더가 fold 아래 대부분을 차지. 

**문제:** 외로운 사람과 귀찮은 사람은 홈 열고 "지금 뭐 할까?"를 찾는데, 즉시 답을 못 받음. 검색바와 그리드 토글은 능동적 탐색 도구지 행동 유도 도구가 아님.

**액션 (H1):** 헤더 바로 아래에 **"한 줄 인풋"을 항상 노출**. "오늘 어떤 하루였어?" placeholder + 한 줄 쓰면 자동으로 일기·대화로 전환. 마찰 0, 진입 즉시 입력 가능. 이게 C 페르소나(귀찮은 사람) 핵심 fix.

### 3.2 Streak 없음

별도 섹션. 이건 다이어리 시장 표준 누락. 

**액션 (S1):** 
- 단순 streak 카운터(연속 X일 일기 작성)부터 시작
- 시각: 헤더 옆에 작은 🔥 X 또는 셸 콘셉트 살려 🐚 X
- 깨지면 1회 "복구" 기회 (Duolingo Streak Freeze 메커니즘) — 너무 가혹하면 역효과
- **다이어리 앱 사용자에게 streak은 게임이 아니라 자기 약속의 증거.** 그래서 시각도 깨끗하게.

**주의:** Streak은 C 페르소나에게는 압박. 첫 7일 안에는 노출 X, 첫 7일 채우면 자동 활성화 + 옵트아웃 가능하게.

### 3.3 "어제의 나"가 매일의 hero가 아님

코드에 `10-home/09-yesterday-change-hint.js`가 있지만 실사용에서 잘 안 보임. 

**액션 (Y1):** 매일 첫 진입 시 **"24시간 전 너의 일기에서 ___을 발견했어"** 같은 hero 카드를 1순위로 노출. AI가 어제 일기·대화에서 한 문장을 골라 다시 비춰주는 mirror 효과. 사용자가 "어 그랬지" 하면서 들어오게 됨.

- 비용: 어제 데이터에서 1줄 추출하는 LLM 호출 (cheap)
- 효과: Day 2~30 retention의 hero. "오늘 들어가야 하는 이유"가 매일 자동 생성됨.

### 3.4 Hook (AI 능동 메시지) cooldown 18시간이 너무 김

`10-home/13-hook-generator.js`가 잘 만들어진 시스템이지만 18시간 cooldown은 신규 사용자에게 너무 가혹.

**액션 (HK1):** 
- Day 0~7: cooldown 6시간으로 단축
- Day 8~30: 12시간
- Day 30+: 현재 18시간
- 사용자 응답률(answered=true 비율)이 일정 임계 이하면 자동 cooldown ↑ (스팸 방지)

## 6. Sentry 데이터로 검증해야 할 가설

이번 진단은 정성 분석. 다음 가설을 Sentry/분석 데이터로 검증하면 우선순위가 더 날카로워짐:

1. **✨ 칩 클릭률** — 깨달음을 저장한 사용자 중 ✨ 칩을 24h 안에 클릭한 비율. K1·K2의 타깃 지표.
2. **나 탭 평균 체류 시간** vs **다른 탭** — M1 시작 전 baseline.
3. **첫 진입 → 첫 일기/대화까지의 drop-off** — 코드 주석에 "84% 가입 후 첫 chat drop" 언급 있음. 실제 수치 확인 후 H1 우선순위 확정.
4. **D1/D7/D30 retention 곡선** — 페르소나별 분리 가능한지(예: 게스트 vs 가입 vs 구독). 다이어리 vs 외로운 vs 귀찮은 페르소나 식별 가능한지.
5. **Hook 메시지 응답률** — 18h cooldown 변경 전 baseline.
