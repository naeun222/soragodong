# `/introduce` 페이지 — 피드백 정리

> 검토 시점: 2026-05-04
> 대상: `https://soragodong.com/introduce/` (저장본 기준)

## 한 줄 요약

페이지 자체는 잘 짜여졌다. 카피톤·신뢰 신호·차별화 포지셔닝까지 다 들어가 있다. 다만 **"끝까지 안 읽는 사람"을 위한 장치가 부족**해서 conversion funnel 후반부가 새고 있을 가능성이 크다.

---

## 잘하고 있는 것

### 1. Hero 의 trust line
"🔒 안심해 — 본인만 열람 가능 · AI 학습·재활용 절대 X · 30일 자동 삭제"
일기 앱에서 첫 0.5초에 박아야 할 메시지를 정확히 박았다. 이거 빠진 일기 앱이 너무 많다. 강점.

### 2. The Problem 섹션의 positioning
일기앱 / 명상앱 / ChatGPT / 심리상담 — 4 카테고리 비교가 또렷하다. 사용자가 "그래서 이게 뭔데?" 라는 물음에 빠르게 답이 와닿는다.

### 3. Feature 1 의 대화 예시 (페이지 최강 컷)
> 💬 나: "오늘 또 연구실 가기 싫어서 인스타 2시간 봤어."
> 🐚 소라고동: "이번 주 세 번째네. 지난번엔 '글쓰기 시작이 제일 힘들다'고 했고, 월경 주기 황체기인 것도 겹쳐. Barkley의 executive function 연구에서 시작 장벽은 외부화로 풀어..."

이 한 컷이 페이지 전체 conversion 의 절반을 책임지고 있을 가능성이 크다. *"와, 진짜 이런 AI가 있어?"* 가 와닿는 순간이다.

### 4. 월간 리뷰 mockup (인터랙티브)
"몰래 잘 쉰 사람" 페르소나 워드 + 디테일 토글 구조. 정적 텍스트 100줄보다 이 인터랙션 하나가 훨씬 강하게 일한다.

### 5. 카피톤
"수치심은 동기부여가 아니야", "경고: 한 달 쓰면 자기 이해가 바뀔 수 있음. 진지하게" — 광고 카피처럼 안 들리고 *사람이 쓴 글* 같다. 이 톤 자체가 차별화이고 retention factor.

---

## 개선 우선순위 (ROI 큰 순)

### 1. CTA 가 페이지 끝에만 있다 — *가장 큰 문제*

사용자가 7 섹션을 끝까지 봐야 "같이 써보기" 가 나온다. 모바일 scroll-depth analytics 가 있다면 50% 이상이 끝에 도달 못 하고 있을 것.

**조치:**
- Hero 바로 아래 inline CTA
- Feature 1 (가장 강한 컷) 직후 CTA
- Philosophy 섹션 끝에 CTA
- 또는 sticky bottom CTA bar (모바일)

세 곳에 CTA 추가만으로도 conversion 이 1.3~2배까지 오를 가능성이 있다 (랜딩 페이지 일반 통계).

### 2. Hero 의 시각적 hook 이 약하다

🐚 이모지 + 텍스트뿐이라 "이 앱이 어떻게 생겼는지" 가 5초 안에 안 보인다.

**조치:** 다음 중 *하나* 라도 hero 영역에 시각화하기:
- 진주(DNA pearl SVG) 한 개를 떠 있게
- 월간 리뷰 카드 mockup 살짝 기울여서
- 채팅 한 컷 (Feature 1 의 대화) 미리보기로

지금은 페이지의 vibe 에만 의지하는 구조. *눈으로 보이는 한 개* 가 hook 강도를 크게 올린다.

### 3. ChatGPT 비교가 위험하다

"ChatGPT 에 상담 — 매번 내가 누군지 설명해야 해. 얘는 어제의 나를 기억 못 해."

ChatGPT memory feature 가 2024년부터 있다. 똘똘한 사용자가 *"이거 거짓말 아냐?"* 라고 의심할 위험.

**조치:** Fair 하게 redirect.
> "ChatGPT memory 도 있지만 — *구조화된 자기 이해 + 패턴 연결 + 14일 숙성 결정*은 다른 문제"

### 4. Feature 2/3 에 진짜 예시가 부족하다

Feature 1 은 강한 dialogue 예시가 있는데:
- Feature 2 (결정의 방) 는 *프로세스 화살표 설명* (`뜨거운 상태 감지 → Odyssey 3경로 작성 → ...`)
- Feature 3 (체크인) 은 예시 자체가 없음

Feature 1 톤으로 통일해야 임팩트가 균등해진다.

**조치:**
- Feature 2: 실제 결정 dialogue 한 컷 ("이 일 그만둘까?" → 14일 후 AI 가 보내는 정렬 메시지)
- Feature 3: 한 달 후 받는 통찰 한 줄 ("너는 수면 5시간 미만 다음날 자책 어휘 3배 늘어")

### 5. 사회적 증거가 0

후기 한 줄, 사용자 숫자, 추천사, 미디어 노출 — 아무것도 없다. 모든 카피를 *개발자가 직접* 말하는 구조라, 끝까지 읽어도 *"근데 진짜 효과 있는 거 맞아?"* 의심이 안 풀린다.

**조치:**
- 베타 사용자 후기 3개 (이름·익명 OK, "한 달 쓴 25세 디자이너" 식)
- 또는 사용 횟수 / 누적 일기 수 / 누적 진주 수 같은 정량 지표
- 또는 추천사 한 줄 ("심리학 박사 ___의 추천")

후기 3개만 있어도 신뢰의 두께가 달라진다.

### 6. "이런 사람한테 좋아" 가 없다

지친 ADHD 직장인 / 번아웃 대학원생 / 기록형 INFP / 자기 이해 좋아하는 사람 — 이 앱의 fit 이 명확하다. 그런데 *명시적인 self-identification 트리거* 가 없어서 사용자가 *"내가 이거 쓸 사람 맞나?"* 를 늦게 판단한다.

**조치:** Philosophy 섹션 앞이나 뒤에 한 블록 추가.
> **이런 사람한테 잘 맞아 ↓**
> · 자기 패턴이 궁금한 사람
> · ADHD / 번아웃으로 일관성 유지가 어려운 사람
> · 결정을 신중하게 하고 싶은 사람
> · 일기는 쓰지만 안 읽히는 게 답답한 사람

Self-identification 이 빠르면 conversion 이 빠르다.

### 7. 가격 / 무료 체험 정보가 없다

끝까지 읽고 "같이 써보기" → 가입 → 결제 요구 → 충격. 무료 체험이 있다면 introduce 끝에 명시. 없다면 가격을 명시하는 게 신뢰 측면에서 더 좋다.

**조치:**
- "처음 14일 무료" 같은 구체 약속이 있다면 CTA 옆에
- 가격 페이지 링크라도 footer 에

### 8. Hero tagline 이 한자 + 추상

"지피지기(知彼知己)에서 시작하는, 일상을 전략적으로 바꾸는 앱"

진성 사용자한테는 멋지지만, 첫 0.5초 인지 부담이 크다. *서브 한 줄로 더 직접적인 카피* 가 받쳐주면 좋다.

**조치 예시:**
- "AI 가 너를 알아가서, 너만의 전략을 짜주는 앱"
- "내 패턴을 AI 가 읽어주는 일기"

지피지기 카피는 그대로 두고, 그 아래 일상어 한 줄을 추가하는 식.

---

## TOP 3 — 시간 없을 때 이것만

가장 ROI 큰 순서로:

1. **다중 CTA 배치** (1번) — 한 시간 작업, conversion 1.3~2x 가능
2. **Feature 2/3 dialogue 보강** (4번) — 임팩트 균등화, 반나절 작업
3. **사회적 증거 3개** (5번) — 베타 사용자 인용, 신뢰의 두께 ↑

나머지는 시간 날 때.

## 데이터로 검증할 만한 가설

이 페이지에 analytics 가 붙어있다면 다음 메트릭을 보면 좋다:

- **Scroll depth** — 50%, 75%, 100% 도달 비율. 50% 미만에서 대부분 이탈하면 1번 (다중 CTA) 이 시급.
- **CTA click-through rate** — 끝까지 읽은 사람 중 몇 %가 "같이 써보기" 누르는지.
- **Heatmap** — Feature 1 dialogue 와 월간 리뷰 mockup 에서 체류 시간이 길 것으로 예상. 검증되면 hero 에 미리보기 옮기는 명분.
- **회원가입 → 첫 체크인 도달율** — 페이지가 *맞는 사람* 을 보내고 있는지 신호.
