## 한 줄 요약

**화면 1·2 (cold open hero) → intakeModal (이미 만들어둔 6스텝) → 나 탭 → 홈** 의 단일 직렬 흐름. 기존 85스텝 코치마크 튜토리얼은 폐기. 게스트와 신규 계정이 동일한 hook 흐름을 받음.

---

## 1. 왜 새로 짜는가

현재 튜토리얼은 **30~40분짜리 박물관 투어**다. 11페이즈, 85스텝, 6개 탭 다 방문, 같은 개념을 두 번 가르침 (DNA 설명 2회, 도서관 yangsaeng 3회). 게스트가 1회 대화로 후킹되는 새 환경에서는 **첫 30초 안에 "어, 이건 다르다"** 가 박혀야 하는데, 현재 첫 화면이 "안녕! 나 소라고동이야 / 친구 톤" 류 자기소개라 차별화 0.

새 안의 핵심은 두 가지:
- **첫 60초가 cinematic 약속** (godong 의 한 마디, 차별점, 첫 질문)
- **나머지는 이미 만들어둔 자산 (intakeModal) 을 그대로 사용** — 새 컴포넌트 거의 안 만듦

---

## 2. 전체 흐름

```
[게스트 / 신규 계정 진입]
        ↓
화면 1: "안녕."                          (다크 hero, ~3초)
        ↓
화면 2: differentiator 3줄                (다크 hero, ~5초)
        ↓
[B안 전환 — 화면 2 가 modal 로 흡수]
        ↓
intakeModal Step 1: 머릿속 첫 한 마디     (기존 그대로)
        ↓
Step 2: AI deepening 질문
        ↓
Step 3: 장문 입력
        ↓
Step 4: "더 알고 싶어" 버튼
        ↓ (게스트는 여기서 회원가입 게이트 — 본인이 이미 설계 완료)
        ↓
Step 5: 4단 분석 (차원/진단/전략)
        ↓
Step 6: "방금 너를 잠깐 들여다봤어 🐚"
        ↓
🔮 나 탭 (분석 결과 카드 — traits/values/patterns)
        ↓ (사용자가 자연스럽게 둘러본 후 본인 의지로 이동)
🏠 홈 (Tier 2 progressive 모드 활성화)
```

## 3. 화면 1 — "안녕."

### 카피 (확정)
```
안녕.
```

### 비주얼
- **배경**: `var(--bg)` (검정 톤, 100vh 풀스크린)
- **godong 아이콘**: `godong.webp`, 96px, 화면 중앙에서 vertical -8% 위치 (시선 안정점)
- **텍스트 위치**: godong 아래로 32px gap
- **폰트**: Gowun Batang serif, 28px, weight 400, letter-spacing 0.02em
- **색**: `var(--text)`

### 모션
- 0ms: 화면 검정으로 진입
- 200~1000ms: godong 페이드인 (opacity 0→1, 800ms ease-out)
- 1000ms: "안녕." 타이핑 effect 시작 — 글자당 120ms (안+녕+. = 약 360ms)
- 1360ms 이후: 1.2초 정적
- 2560ms~: 우하단에 ✦ 아이콘 페이드인 (탭 가능 시그널, opacity 0.3 정도)

### 인터랙션
- 화면 어디든 탭 → 화면 2 로 전환
- ESC / 뒤로 / 건너뛰기 X (cold open 은 못 건너뜀)
- 단, 우상단에 작은 X (닫기) — 게스트는 일반 채팅으로, 신규는 빈 홈으로 빠짐
- "탭해서 계속" 같은 explicit hint 절대 X — ✦ 점멸로만

### 톤 의도
- 가벼운 인사가 아니라 **"한 박자 늦은 다정함"**. 인사 끝 마침표가 결정적.
- 첫 3초가 정적/godong 만 보이는 이유: **이 앱은 빨리 답하는 앱이 아니다** 라는 톤 셋업.

## 4. 화면 2 — Differentiator

### 카피 (확정)
```
다른 AI랑 나는 좀 달라.
너를 기억해.
그리고 시간이 지나면 — 다시 물어봐.
```

### 비주얼
- 화면 1 의 godong + 검정 배경 그대로 유지 (시각 연속성)
- 텍스트는 화면 1 의 "안녕." 위치를 그대로 차지하며 fade-replace
- 폰트: Gowun Batang serif
  - 1행 "다른 AI랑 나는 좀 달라." — 22px, weight 400
  - 2행 "너를 기억해." — 24px, weight 500, 미세 accent (var(--accent))
  - 3행 "그리고 시간이 지나면 — 다시 물어봐." — 22px, weight 400
- 줄 간격: 16px

### 모션 — 한 줄씩 cascade fade in
- 0ms: 1행 fade in (300ms)
- 600ms: 2행 fade in (300ms) ← 임팩트 라인. 약간 지연.
- 1200ms: 3행 fade in (300ms)
- 1500ms 이후: 1.5초 정적
- 3000ms~: ✦ 페이드인 (탭 시그널)

### 인터랙션
- 탭 → B안 전환 시작
- 탭 안 함 시 ✦ 가 천천히 호흡 (3초 주기 opacity 0.3↔0.6)

### 톤 의도
- 이 3줄이 **앱 USP 의 압축본**. 14일 숙성도, 도서관도, 모델 학습도 다 "기억하고 다시 묻는다" 의 변주.
- 2행 "너를 기억해." 가 강조되는 이유: 다른 모든 약속이 여기서 파생됨. 한 박자 살짝 위.

## 5. 전환 — 화면 2 → intakeModal (B안)

화면 2 의 텍스트와 godong 이 **modal 안으로 흡수**되는 연출. 본인이 B 채택했으므로 두 가지 변형 중 추천하는 쪽을 박아둠.

### 두 변형 비교

**B-1: 화면 2 의 마지막 줄 ("그리고 시간이 지나면 — 다시 물어봐.") 만 modal 상단 헤더로 영구 안착**
- 약속이 항상 시야에. 무거움.
- 입력 영역이 시각적 제2의 영역으로 밀림.

**B-2: 화면 2 텍스트 전체 fade out + modal step 1 prompt 가 같은 폰트·같은 위치로 등장 — 같은 화자가 다음 말을 이어가는 느낌** ⭐ 추천
- modal 안에서 사용자 입력 영역이 주인공.
- "흡수" 가 텍스트 그 자체가 아니라 **톤·위치·스타일의 흡수**.
- 화면 2 의 godong 만 시각적으로 살아남아 헤더 좌상단으로 이동.

### B-2 타임라인 (확정)

| 시각 | 사건 |
|---|---|
| 0ms | 사용자 화면 2 탭 |
| 0~400ms | 화면 2 텍스트 3줄 fade out (opacity 1→0) |
| 0~300ms | 다크 배경이 `var(--surface)` 로 fade (top-down radial) |
| 100~700ms | godong 96px → 36px 축소, 화면 중앙 → 좌상단 (top:24px, left:24px) 이동 |
| 300~650ms | intakeModal 컨테이너 slide up (translateY 100%→0, ease-out) |
| 500~700ms | progress dots 6개 fade in |
| 650~850ms | step 1 prompt 텍스트 fade in (Gowun Batang, 18px — 화면 2 톤 그대로) |
| 850~1000ms | mic 추천 배너 + textarea + send 버튼 순차 fade in |
| 1000ms+ | 사용자 입력 가능 |

전체 1초. cinematic 하지만 답답하지 않음. **godong 이 좌상단에 작게 살아있는 게 핵심** — 화자가 계속 같은 사람임이 시각적으로 박힘.

## 6. intakeModal 6 스텝

**현재 `04-annual-review-phase1/08-intake-modal-flow.js` 의 구현을 그대로 사용.** 새로 만들지 않음. 단, 두 가지 미세 수정 후보가 있음 (본인 결정 필요).

### 6.1 Step 1 prompt 카피 — **결정 필요**

화면 2 와 직접 이어져야 하므로 톤이 바뀌면 단절됨. 본인이 이전에 거부한 카피들 (`그래서 다시 물어볼 첫 한 가지를 받아볼게` 류 — 약속 중복) 제외. 후보:

| 옵션 | 카피 | 톤 / 인상 |
|---|---|---|
| **A (현재 그대로)** | 요즘 마음에 자주 떠오르는 거, 한 마디부터 시작해도 OK. | 시인적·약간 추상. 회피 가능. |
| **B** | 머릿속에서 안 떠나는 거. 한 줄도 OK. | 직감적. 누구나 정확히 한 가지 떠올라. 후킹 강도 ↑. |
| **C** | 지금 머릿속에 뭐가 있어? | 친구 톤. 가볍고 자연스러움. |
| **D** | 풀고 싶은 거, 하나만. | 행동 지향. 문제해결 frame. |

**미감 / 본인 톤 직감 기준으로 골라줘.** 내가 추천 안 하는 이유: "AI 같은 카피" 직감 기준이 본인이 더 정확함. 위 4개 다 무난한데, 화면 2 와 가장 자연스러운 단절 없는 건 B 또는 C.

### 6.2 Step 6 끝 라인 — **결정 필요**

현재 Step 6:
```
🐚 (큰 emoji)
방금 너를 잠깐 들여다봤어 🐚
✨ 나 탭 가보면 — 내가 방금 너를 어떻게 봤는지 나와있어. 이따 확인해봐!
```

여기에 "14일 약속" 한 줄을 더할지:

> "그리고 — 너가 말한 그 한 가지, 14일 후에 내가 다시 물어볼게."

화면 2 의 약속을 모달 끝에서 한 번 더 박는 효과. retention loop 가 cold open 과 close loop 양쪽에서 reinforced. 단, 이 라인이 **godong 톤** 인지 **AI 톤** 인지는 본인 직감으로 판단.

대안: 추가 안 함. 화면 2 약속은 화면 2 에서 끝. 모달은 첫 자기 관찰 결과로 마무리.

### 6.3 그 외 — 손대지 말 것

- 6 dot progress
- 🎤 mic 배너 ("말로 풀어봐 — 적극 추천!")
- 예시 칩 (클릭해서 textarea 채움)
- AI deepening (15자 미만 자동 follow-up)
- ESC / 외부 클릭 차단
- Step 5 의 4단 분석 카드 (차원/진단/전략)

전부 잘 됨. 미감 좋고 후킹 강함. 그대로 사용.

## 8. 게스트 vs 신규 계정 분기

본인이 이미 설계 완료. 이 문서에서는 **진입점만 명시**:

- **게스트 진입**: 화면 1 → 2 → intakeModal Step 1~3 까지 동일. **Step 4 ("더 알고 싶어" 버튼) 클릭 시점에 회원가입 게이트** (본인 설계). 게이트 통과 시 Step 5 로 이어짐. 거부 시 그 시점까지의 `intakeWorry` localStorage 보관 → 추후 가입 시 마이그레이션 (이미 `worryData` 격리 인프라 있음).
- **신규 계정 진입**: 화면 1 → 2 → intakeModal 6스텝 풀로 → 나 탭.

## 9. 기존 80스텝 처리

폐기 또는 이동. 구체적으로:

| 현재 위치 | 새 처리 |
|---|---|

**Tier 3 = 설정 안 "🐚 가이드 둘러보기" 메뉴**. 옵셔널. 카드형 모듈 6개:
1. 체크인 깊이 쓰기

5. DNA / 진화 / 결정화
6. 실행 탭 + 몰입 단축어

각 모듈 30~90초. 강제 X. 호기심 driven.

`miniTutorialsSeen` / `progressiveUnlockLevel` / `tutorialVersion` 상태 필드는 이 분리에 그대로 활용.

## 10. 톤 가이드 (godong 카피 원칙)

본인 직감 ("AI 같다" 판별) 기준으로 정리:

### 피할 것
- **명령조 / 지시조** — "~해", "~해야 해", "이제 ~로 가" → 카피의 주체가 사용자에게 명령하는 AI 가 됨
- **자기보고조** — "확인했어", "처리했어", "접수했어" → godong 이 자기 행동을 알리는 봇 톤
- **친절 포장** — "도와드릴게요" 류. 친절도 commodity.
- **약속 중복** — 한 번 한 약속 재진술 ("다시 물어볼게" → "그래서 다시 물어볼 첫 한 가지" 같은)

### 표준
Step 6 의 `방금 너를 잠깐 들여다봤어 🐚` — **이게 godong 톤의 표준**. 짧고 감각적이고, godong 이 자기 행동을 commodity 처럼 보고하지 않고, 사용자 곁에서 함께 있는 톤.

### 디테일
- 짧은 문장. 한 줄에 한 호흡.
- 구두점 신중. 줄임표 절제 (`...` 남용 금지).
- 마침표가 결정적. 마침표 한 개로 무게 잡기.
- 반말 — 친구 톤이지만 가볍지 않게.

## 11. 미감 가이드 (UI/visual 원칙)

### 타이포그래피
- **Cold open (화면 1·2)**: Gowun Batang serif, 22~28px, letter-spacing 0.02em
  - serif 의 이유: 평범한 시스템 산세리프와 차별. 한국어 serif 가 줄 수 있는 "한 박자 늦은 정중함" 이 godong 톤과 정확히 맞음.
- **Modal 본문**: 시스템 산세리프 (intakeModal 현재 폰트 그대로)
- **사이즈 ladder**: 28 → 24 → 22 → 18 → 14 → 12

### 컬러
- Cold open 배경: `var(--bg)` (검정)
- Cold open 텍스트: `var(--text)` (밝음), 임팩트 라인만 `var(--accent)`
- Modal 배경: `var(--surface)` (밝음)
- godong icon: 그대로 (recolor X)
- ✦ 글리프 색: accent gradient (이미 앱 톤 일관성)

### 모션
- 모든 fade: 300~800ms ease-out
- 텍스트 타이핑: 글자당 80~120ms (화면 1 만)
- 줄 cascade fade: 줄당 600ms gap (화면 2)
- Slide up: 350ms ease-out
- 임팩트 라인 후 정적: 1.2~1.5초 (관용 필요)

## 부록 — 판단 보존

### 왜 화면 1·2 가 cinematic 다크인가
일반 앱 톤 (밝은 surface, 친절 카피) 과 의도적으로 다르게 시작. 사용자가 **"어, 다른 거다"** 를 첫 3초에 인지하게. Step 6 의 godong + Gowun Batang serif 톤이 이미 앱 안에 살아있으므로 cold open 도 같은 결로.

### 왜 mic 배너가 후킹에 중요한가
타이핑 vs 음성 = 첫 입력 마찰의 차이가 큼. "🎤 말로 풀어봐 — 적극 추천!" 이 첫 화면에서 음성 사용을 정당화. 한 줄 적기 어려운 사용자가 음성으로 한 마디 흘려도 OK 라는 진입장벽 0 시그널. **이 한 배너가 게스트 conversion 의 큰 부분을 책임짐.**

### 왜 80스텝을 폐기해도 되는가
85스텝 중 60% 가 "단순 설명" 또는 "탭 안내" — Tier 2 progressive 트리거 / Tier 3 옵셔널 모듈로 옮기면 학습 효율 동일하거나 우월. 강제 학습은 첫 사이클 완수 (= intakeModal 6스텝) 까지로 충분. 그 이상은 사용자 호기심에 맡김.
