# 백엔드 프록시 — DRAFT (작동 X)

**현재 상태**: 템플릿 / reference 만. 활성 X.

## 왜 멈췄나

프록시 단독으로 박으면 너 (개발자) 가 모든 사용자 API 비용 부담. 청구 인프라 X = 의미 없음.

본격 활성화는 다음이 다 박힌 후:

1. **결제 시스템** (PG사 통합 — 포트원 등)
2. **사업자등록 + 통신판매업 신고**
3. **이용약관 + 개인정보처리방침** (변호사 검수)
4. **사용량 cap / 한도** (사용자별 월 한도 제어)
5. **요금제 정의** (정액제 / 사용량 기반 / 하이브리드)
6. **`soragodong_usage` 테이블 생성** (Supabase)
7. **Anthropic 서버용 API 키** 발급 (사용자 본인 키와 분리)
8. **국외이전 동의** (사용자한테 명시 동의 받음)

## 그때까지

현재 흐름 유지: **사용자가 자기 Anthropic API 키 입력 → 클라이언트에서 직접 호출**.

장점:
- 너 비용 0
- 사용자 데이터 = 사용자 ↔ Anthropic 직접 (proxy 안 거침)
- 베타 단계 자유롭게

단점:
- 사용자가 API 키 발급할 줄 알아야 (개발자/얼리어답터 친화)
- 사용자 데이터가 사용자 본인 키로 결제됨 (claude.ai 결제와 별개)

## 활성화 시점

베타 5–10명 받고 정식 출시 / 수익화 시작 직전.

## 파일

- `_lib/auth.ts` — Supabase JWT 검증 헬퍼
- `_lib/usage.ts` — 사용량 추적 + 가격 계산 (Claude Sonnet 4.6 기준)

이 두 파일은 활성화 시 `api/` 로 옮기고 `chat.ts`, `usage.ts` 등 endpoint 추가.
